git status - текущая состояние проекта git diff - показывает различия в текущем и сохраненном состоянии git diff --staged показывает проиндексированные изменения git add <имя фала> - сохранение текущий изменений. .(Точка) - сохранить все изменения в файле git commit -m "текст" - добавление описание к сохраненному состоянию git commit --amend -m "Правильное сообщение коммита" - заменяет сообщение в коммите git commit --amend --no-edit git checkout <имя файла> - откатить изменения git checkout <хеш коммита> <имя файла>- откатить изменения коммита git checkout <хеш коммита> - перемещаемся в текущий коммит git checkout -b <метка, название> <хеш коммита> - Перемещаемся в текущий коммит и присваеваем ему название(метку) git checkout -b <название новой ветки> - создаем новую ветку git cat-file -p <хеш коммита> - показывает коммит в человеческом виде git log - выводить историю изменений. Параметр -1 показывает последний коммит. -2 - показывает 2 предыдущих и т.д. git log --oneline - выводить коммиты в одну строку git log --oneline --all - показывает все коммиты git log --oneline --all --graph - показывает все коммиты графически git reset HEAD <имя фала> - отменяет изменения в файле и отменяет индексацию git show <Хеш коммита> - показывает изменения в коммите git rm <имя фала> -удаляет файл из коммита и удаляется из папки проекта git rm --cached <имя или путь к файлу> - удаляет файл из коомита и не удаляет из папки git help <команда> - показывает справку команды git merge <название ветки> -m "сообщение коммита" - слияние двух веток ssh -T -i ~/.ssh/<название приватного ключа> git@github.com - доступ по ssh на сервер git remote -v проверка репозиториев ssh-keygen -t rsa -b 4096 -C "емаил адрес" Генерация открытого и закрытого ключа для доступа на сервер git branch - m <Имя ветки> - переименовывает, удаляет и создает ветки git fetch origin - получить изменения в репозитории
